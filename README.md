## 🚀 Overview

A **production-ready** FastAPI application for secure VPS management and integration. Designed specifically for **Oracle VPS (ARM64)** and **MCP (Model Context Protocol)** integration with **Poke**. Features comprehensive system monitoring, Docker management, secure file operations, and process control with enterprise-grade security.

## ✨ Key Features

- **🔐 Enterprise Security**: GitHub OAuth + JWT authentication, rate limiting, encryption
- **📊 System Monitoring**: Real-time CPU, memory, disk, network statistics
- **🐳 Docker Management**: Complete container lifecycle control (start, stop, logs, stats)
- **📁 Secure File Operations**: Read, write, delete with path validation
- **⚙️ Process Management**: Monitor and control system processes
- **🏗️ ARM64 Support**: Optimized for Oracle Cloud ARM instances
- **🔌 MCP Integration**: Ready for Poke and other MCP clients
- **📈 Production Ready**: Rate limiting, logging, error handling, health checks
- **📚 Auto Documentation**: Interactive Swagger UI and ReDoc
- **🚀 Easy Deployment**: One-command installation script

## ⚡ Quick Start

### Option 1: Automated Installation (Recommended)

```bash
# SSH into your Oracle VPS
ssh user@your-vps-ip

# Download and run installer
curl -fsSL https://raw.githubusercontent.com/jessiorg/vps-integration-api/main/scripts/install.sh | sudo bash
```

The installer will:
- ✅ Detect architecture (x86_64/ARM64)
- ✅ Install all dependencies
- ✅ Configure Docker
- ✅ Set up Nginx reverse proxy
- ✅ Create systemd service
- ✅ Configure firewall

### Option 2: Manual Installation

```bash
# Clone repository
git clone https://github.com/jessiorg/vps-integration-api.git
cd vps-integration-api

# Configure environment
cp .env.example .env
nano .env  # Add your GitHub OAuth credentials

# Deploy with Docker Compose
docker-compose up -d

# Or use installation script
sudo chmod +x scripts/install.sh
sudo ./scripts/install.sh
```

### Initial Configuration

1. **Create GitHub OAuth App**:
   - Go to: https://github.com/settings/developers
   - New OAuth App
   - Homepage URL: `https://your-vps-domain.com`
   - Callback URL: `https://your-vps-domain.com/auth/github/callback`

2. **Update `.env` file**:
   ```bash
   sudo nano /opt/vps-integration-api/.env
   ```
   
   Set these values:
   ```env
   GITHUB_CLIENT_ID=your_client_id_here
   GITHUB_CLIENT_SECRET=your_client_secret_here
   GITHUB_REDIRECT_URI=https://your-vps-domain.com/auth/github/callback
   ALLOWED_USERS=your_github_username
   SECRET_KEY=<auto-generated>
   ```

3. **Start the service**:
   ```bash
   sudo systemctl start vps-integration-api
   sudo systemctl status vps-integration-api
   ```

4. **Access the API**:
   - API: http://your-vps-ip
   - Docs: http://your-vps-ip/docs
   - Health: http://your-vps-ip/health

## 🔌 MCP Integration

### Connect with Poke

The API is designed for seamless integration with MCP clients like Poke. See detailed guide: [docs/MCP_INTEGRATION.md](docs/MCP_INTEGRATION.md)

**Quick MCP Setup:**

1. **Get Access Token**:
   ```bash
   # Visit in browser
   https://your-vps-domain.com/auth/github
   # Copy the JWT token from response
   ```

2. **Configure MCP Server** (`~/.config/poke/mcp-servers.json`):
   ```json
   {
     "vps": {
       "command": "node",
       "args": ["vps-mcp-server.js"],
       "env": {
         "VPS_API_URL": "https://your-vps-domain.com",
         "VPS_API_TOKEN": "your_jwt_token"
       }
     }
   }
   ```

3. **Use with Poke**:
   ```
   "Hey Poke, check the system status on my VPS"
   "Poke, restart the nginx container"
   "Show me the logs from my app container"
   ```

## 📚 API Documentation

### System Monitoring

| Endpoint | Description |
|----------|-------------|
| `GET /api/v1/system/info` | System information (hostname, platform, CPU, memory) |
| `GET /api/v1/system/cpu` | CPU usage and frequency |
| `GET /api/v1/system/memory` | Memory usage statistics |
| `GET /api/v1/system/disk` | Disk usage by path |
| `GET /api/v1/system/network` | Network I/O statistics |
| `GET /api/v1/system/uptime` | System uptime |

### Docker Management

| Endpoint | Description |
|----------|-------------|
| `GET /api/v1/docker/containers` | List all containers |
| `GET /api/v1/docker/containers/{id}` | Get container details |
| `POST /api/v1/docker/containers/{id}/start` | Start container |
| `POST /api/v1/docker/containers/{id}/stop` | Stop container |
| `POST /api/v1/docker/containers/{id}/restart` | Restart container |
| `DELETE /api/v1/docker/containers/{id}` | Remove container |
| `GET /api/v1/docker/containers/{id}/logs` | Get container logs |
| `GET /api/v1/docker/containers/{id}/stats` | Get container stats |
| `POST /api/v1/docker/containers` | Create new container |
| `GET /api/v1/docker/images` | List images |

### File Operations

| Endpoint | Description |
|----------|-------------|
| `GET /api/v1/files/read?path={path}` | Read file contents |
| `POST /api/v1/files/write` | Write to file |
| `DELETE /api/v1/files/delete?path={path}` | Delete file |
| `GET /api/v1/files/list?path={path}` | List directory |
| `GET /api/v1/files/exists?path={path}` | Check file existence |

### Process Management

| Endpoint | Description |
|----------|-------------|
| `GET /api/v1/processes` | List running processes |
| `GET /api/v1/processes/{pid}` | Get process details |
| `DELETE /api/v1/processes/{pid}` | Kill process |
| `GET /api/v1/processes/search/by-name?name={name}` | Search processes |

### Authentication

| Endpoint | Description |
|----------|-------------|
| `GET /auth/github` | Initiate OAuth flow |
| `GET /auth/github/callback` | OAuth callback |
| `GET /auth/verify` | Verify JWT token |
| `POST /auth/logout` | Logout |

**Interactive Documentation**: Visit `http://your-vps-ip/docs` for Swagger UI

## 🔒 Security

### Multi-Layer Security

1. **Authentication**: GitHub OAuth 2.0
2. **Authorization**: JWT tokens (HS256)
3. **Rate Limiting**: 60 requests/minute (configurable)
4. **Path Validation**: Whitelist/blacklist for file access
5. **Security Headers**: HSTS, CSP, X-Frame-Options, etc.
6. **Encryption**: HTTPS/TLS enforced
7. **User Allowlist**: Restrict access by GitHub username

### Security Best Practices

```bash
# 1. Use HTTPS only
sudo certbot --nginx -d your-domain.com

# 2. Restrict file access
ALLOWED_PATHS=/data,/var/log,/etc/nginx
FORBIDDEN_PATHS=/etc/passwd,/etc/shadow,/root

# 3. Limit user access
ALLOWED_USERS=trusted_user1,trusted_user2

# 4. Monitor API logs
sudo journalctl -u vps-integration-api -f

# 5. Regular backups
sudo /opt/vps-integration-api/scripts/backup.sh
```

## 🚀 Deployment

### Production Deployment

```bash

docker-compose -f docker-compose.yml -f docker-compose.prod.yml up -d

# 2. Set up SSL
sudo certbot --nginx -d your-domain.com

# 3. Enable firewall
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
```

## 💡 Examples

### Example 1: System Health Check

```python
import requests

API = "https://your-vps.com/api/v1"
TOKEN = "your_jwt_token"
headers = {"Authorization": f"Bearer {TOKEN}"}

# Get all system stats
cpu = requests.get(f"{API}/system/cpu", headers=headers).json()
memory = requests.get(f"{API}/system/memory", headers=headers).json()
disk = requests.get(f"{API}/system/disk", headers=headers).json()

print(f"CPU: {cpu['usage_percent']}%")
print(f"Memory: {memory['percent']}%")
print(f"Disk: {disk['percent']}%")
```

### Example 2: Restart Container

```bash
curl -X POST \
  -H "Authorization: Bearer $TOKEN" \
  https://your-vps.com/api/v1/docker/containers/nginx/restart
```

### Example 3: Deploy Application

```python
import requests

API = "https://your-vps.com/api/v1"
headers = {"Authorization": f"Bearer {TOKEN}"}

# Write app code
requests.post(f"{API}/files/write", headers=headers, json={
    "path": "/data/app/index.js",
    "content": "console.log('Hello World');"
})

# Start container
requests.post(f"{API}/docker/containers", headers=headers, json={
    "image": "node:18",
    "name": "myapp",
    "command": "node /app/index.js",
    "volumes": {"/data/app": {"bind": "/app", "mode": "ro"}}
})
```

## 🔧 Troubleshooting

### Service won't start

```bash
# Check logs
sudo journalctl -u vps-integration-api -n 100

# Verify Docker is running
sudo systemctl status docker

# Check port availability
sudo netstat -tulpn | grep 8000
```

### Authentication errors

```bash
# Verify token
curl -H "Authorization: Bearer $TOKEN" \
  https://your-vps.com/auth/verify

# Check allowed users
grep ALLOWED_USERS /opt/vps-integration-api/.env
```

### Docker access issues

```bash
# Check Docker socket permissions
ls -l /var/run/docker.sock

# Restart service
sudo systemctl restart vps-integration-api
```
